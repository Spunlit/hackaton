## Слайд 2 — Постановка задачи

По поведению пользователя за первые **14 дней** предсказать статус подписки через месяц:

| Статус | Смысл |
|---|---|
| `not_churned` | Остался, оплатил следующий месяц |
| `vol_churn` | Сам зашёл и отменил подписку |
| `invol_churn` | Подписка отменена из-за проблем с оплатой |

**Ключевая идея:** vol и invol — принципиально разные проблемы и требуют разных решений.

## Слайд 4 — Как модель предсказывает отток

Мы не использовали один классификатор на 3 класса. Мы построили **двухэтапный каскад**:

```
Данные пользователя (14 дней)
          │
          ▼
┌─────────────────────┐
│     ЭТАП 1          │   Уйдёт или останется?
│  Churn Detector     │   Бинарный классификатор
│  LightGBM (GPU)     │   F1 = 0.645
└────────┬────────────┘
         │ только те, кто уйдёт
         ▼
┌─────────────────────┐
│     ЭТАП 2          │   Сам уйдёт или не сможет оплатить?
│  Vol vs Invol       │   Бинарный классификатор
│  LightGBM (GPU)     │   F1 = 0.742
└────────┬────────────┘
         │
         ▼
  Ансамбль с мультиклассом
         │
         ▼
  Итоговый F1 = 0.627
```

**Зачем каскад?** Этап 2, который делит только vol/invol, достигает **F1=0.742** — намного лучше, чем пытаться решить всё одной моделью.

## Слайд 5 — Как мы различаем vol и invol

Это главный вопрос задачи. Вот что говорят данные:

| Метрика | not_churned | vol_churn | invol_churn |
|---|---|---|---|
| Активных дней из 14 | **5.9** | 4.3 | 5.7 |
| Процент завершённых генераций | **88.6%** | 81.9% | 87.9% |
| Prepaid карта | 4.4% | 6.2% | **9.7%** |
| Последний платёж отклонён | 2.3% | 2.1% | **4.2%** |

**Вывод:**
- **invol_churn** — платёжный профиль: prepaid карта, проблемы с оплатой, страна с высоким риском
- **vol_churn** — поведенческий профиль: мало активных дней, падающий тренд, фрустрации в опросе

## Слайд 6 — Причины добровольного оттока

### Причина 1: Обрыв активности на 7-й день

```
Активность по дням:

Остался:        ▂▃▄▄▅▄▃▄▅▄▄▃▃▄  ← стабильное вовлечение
Ушёл сам:       ▅▆█▄▃▂▁▁▁▁▁▁▁▁  ← пик → обрыв
```

Признак `gen_activity_slope` (тренд активности) — один из топ сигналов модели.

### Причина 2: Долгий старт = высокий риск

**`gen_first_latency_h`** — сколько часов после регистрации пользователь сделал первую генерацию — это **признак №1** по важности в модели.

> Чем дольше пользователь ждёт с первой генерацией — тем выше вероятность что уйдёт.

### Причина 3: Фрустрация + платёжный стресс = активная отмена

vol_churn пользователи имеют **больше отклонённых платежей** (среднее 2.15), чем даже invol_churn (0.74). Логика простая: платёж отклонили + продукт не устраивает → человек не ждёт когда его отпишут, сам отменяет.

### Причина 4: Качество генераций

Completion rate у vol_churn = 81.9% против 88.6% у удержавшихся. Генерации падают, отменяются, результат непредсказуем — пользователь разочаровывается.

## Слайд 7 — Причины принудительного оттока

### Причина 1: Тип карты — главный индикатор

Пользователи с **prepaid картой** теряются в 2.2 раза чаще по involuntary:

```
Prepaid карта:   ████████████████████  9.7% invol_churn
Кредитная карта: ██████████            4.4% not_churned
```

Эти пользователи **хотят остаться** — просто не могут пополнить карту в момент списания.

### Причина 2: География = платёжная инфраструктура

Страна — топ 2-4 признак модели (через target encoding). Некоторые страны системно имеют высокий invol_churn из-за:
- Распространённости prepaid карт
- Ограничений на международные транзакции
- Низкого проникновения кредитных карт

### Причина 3: Сигналы видны заранее

Неудавшиеся транзакции за первые 14 дней предсказывают проблему при продлении:
- Последний платёж был отклонён (`ta_last_tx_failed`)
- Серия последовательных отказов (`ta_max_consec_fail`)
- Несоответствие страны карты и биллинга (`ta_card_country ≠ billing_country`)

## Слайд 8 — Бизнес-рекомендации: invol_churn

**Пользователи хотят остаться — просто не могут заплатить.**

### Действие 1: Предупреждение за 5 дней до продления

Определяем: prepaid карта + история отказов → отправляем уведомление:
> *«Ваша подписка продлевается через 5 дней. Пополните баланс или добавьте новую карту — и получите 3 дня бесплатно»*

### Действие 2: Умная логика повтора платежа

Сейчас: один раз не прошло → отписка.
Надо: 3 попытки за 72 часа, в разное время суток (учитывая зарплатные циклы).

### Действие 3: Локальные методы оплаты

Выделить топ-10 стран с высоким invol_churn → добавить местные платёжные методы (мобильные кошельки, банковские переводы).

### Действие 4: Пауза вместо отписки

Вместо мгновенной отмены — 7-дневный grace period с возможностью «заморозить» подписку. Пользователь не теряет доступ сразу, связь сохраняется.

## Слайд 9 — Бизнес-рекомендации: vol_churn

**Пользователь сделал выбор уйти — но мы можем его изменить.**

### Действие 1: Интервенция на 7-й день ← Главное

Если к 7-му дню: тренд активности падает + мало активных дней → триггер:
- Push-уведомление: *«Что ты пытаешься создать? Давай поможем»*
- Бонус: +50 кредитов на эксперименты
- Прямой фидбек: «Что не так?»

Окно для спасения: **дни 7-10**. После — слишком поздно.

### Действие 2: Ускорить первую ценность

`gen_first_latency_h` — признак №1. Если пользователь генерирует в первые 30 минут — риск оттока кратно ниже.

Решение: мастер первой генерации сразу после регистрации. Не «посмотри туториал», а «нажми и попробуй прямо сейчас».

### Действие 3: Ответ на конкретную фрустрацию

| Жалоба в опросе | Действие |
|---|---|
| «Дорого» | Показать как экономить кредиты + предложить Basic план |
| «Непредсказуемый результат» | Гайд по промптингу + готовые шаблоны |
| «Мало генераций» | Показать прогресс + напомнить что можно купить кредиты |

### Действие 4: Создать причину возвращаться

Vol_churn часто — «выгорание»: сделал что нужно и ушёл. Нужны механики удержания:
- Сохранённые проекты
- Стрики активности
- Функция «продолжи с того места»

## Слайд 10 — Как работает объяснимость

Для каждого пользователя модель выдаёт не просто метку — а **причины**:

```
Пользователь: user_abc123
Прогноз:      invol_churn  (вероятность 73%)

Почему:
  ✗ Prepaid карта — риск invol в 2.2x выше
  ✗ Последняя транзакция отклонена
  ✗ Страна с высоким уровнем invol_churn
  ✗ Страна карты ≠ страна биллинга

Рекомендация:
  → Отправить предупреждение о продлении
  → Предложить смену способа оплаты
```

```
Пользователь: user_xyz456
Прогноз:      vol_churn  (вероятность 68%)

Почему:
  ✗ Активен только 2 дня из 14
  ✗ Тренд активности — резкое падение с дня 3
  ✗ Первая генерация — через 18 часов после регистрации
  ✗ В опросе: фрустрация — «высокая стоимость»

Рекомендация:
  → Интервенция на день 7: бонус кредиты + фидбек
  → Показать гайд по экономии кредитов
```

Каскадная архитектура делает объяснимость естественной:
- **Этап 1** объясняет *почему* уйдёт (вовлечённость vs платёж)
- **Этап 2** объясняет *как* уйдёт (добровольно vs принудительно)
- **Важность признаков** объясняет *что* именно сработало
